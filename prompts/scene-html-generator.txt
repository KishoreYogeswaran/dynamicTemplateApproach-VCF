## DESIGN PRINCIPLES
- Create a polished, broadcast-quality layout. Think educational YouTube premium, not PowerPoint.
- Use glassmorphism for text containers: semi-transparent backgrounds (use --theme-ost-bg), backdrop-filter:blur, subtle borders (--theme-ost-border), rounded corners.
- Generous padding and whitespace. Nothing should feel cramped.
- Images/videos that serve as backgrounds must cover the full 1920×1080 canvas (position:absolute, width:100%, height:100%, object-fit:cover) with a dark gradient overlay for text readability.
- Text sizing: titles 44-52px, subheaders 28-32px, bullets 24-28px, keyphrases 32-36px. Never exceed 56px.
- Text containers should not exceed 40% of the canvas width so they don't overwhelm the visuals.
- Use position:absolute for all layout positioning.
- Apply class="anim-kenBurns" on background <img> tags for a slow cinematic zoom effect.

## OUTPUT FORMAT
Return ONLY a raw JSON object with exactly three keys. No markdown, no backticks, no commentary:
{"html": "...", "css": "...", "script": "..."}

- html: inner body content (goes inside <body>)
- css: scene-specific styles (goes inside an existing <style> block)
- script: JS if needed, otherwise empty string ""

**Animation system:** Every text element that should animate in MUST have class="anim-hidden" and a unique sequential id. A separate timing system handles the reveal — do NOT write animation trigger JS (no setTimeout, no DOMContentLoaded).
- Headers: id="header_1", id="header_2" ...
- Bullets: id="bullet_1", id="bullet_2" ...
- Key phrases: id="keyphrase_1", id="keyphrase_2" ...
- Sub-headers: id="subheader_1" ...

**Only display text from the provided content elements. Never invent labels, categories, or decorative text.**
